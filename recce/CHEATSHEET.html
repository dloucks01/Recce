Attack path./bin/recce attackpath -o engchains findings → domain compromise
Credentials./bin/recce creds --add 'dom\u:p' -o engstack creds → spray plan (--plan)

Report./bin/recce report -o engrefold Excel edits + rebuild HTML/MD

Tip  ./bin/recce is a shortcut for python3 -m recce — use whichever runs. Run scans with sudo so nmap can do SYN/OS detection.

Zero ports?  Hosts block ping (firewalled / Windows / AD). Add -Pn to treat every target as up. recce also auto-falls-back to -Pn when discovery gets no responses.

enum/scan take the scope to scan. vulns/db/privesc take targets to work on just that subset of what's already enumerated.

3Speed & filters

  • --fast — top-signal checks only, live % + ETA on big ranges
  • --aggressive — brute / nikto / dir-busting (loud)
  • --only http smb — restrict to named services
  • --unscanned — only ports not yet vuln-scanned
  • --workers N · --profile quick · --host-timeout
  • --resume — pick up an interrupted run
  • --no-probes · --no-searchsploit — skip extras
Env  RECCE_DEBUG=1 prints the full traceback on any crash.

Read-only  The on-target sweeps only read — they enumerate and flag what's escalatable; they never change the box. ingest then determines what's actually priv-escable from what the sweep found.

Every phase is idempotent  Re-running never duplicates rows — it's always safe to run a phase again. Full reference: README.md · deep help: TROUBLESHOOTING.md · walkthrough: QUICKSTART.md.
